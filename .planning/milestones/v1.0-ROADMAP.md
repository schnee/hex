# Roadmap: Hex Layout Toolkit

## Overview

This roadmap delivers a parity-first migration from Streamlit to a React-first experience by shipping complete user workflows in dependency order: pattern generation, wall overlay interaction, precision placement, full routed cutover, and reliability polish. Each phase is defined by user-observable outcomes that preserve validated backend behavior while improving front-end interaction quality.

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: React Pattern Generation** - Users can configure, generate, select, and export patterns in React.
- [ ] **Phase 2: Overlay Interaction Foundation** - Users can upload wall images and directly manipulate overlay objects.
- [ ] **Phase 3: Precision Placement Controls** - Users can place overlays accurately with live dimensions and canvas navigation.
- [ ] **Phase 4: End-to-End React Workflow Cutover** - Users can complete the full routed workflow in React with API-contract parity.
- [ ] **Phase 5: Reliability & UX Stability** - Users get consistent feedback and stable interactions across core workflows.
- [ ] **Phase 6: Generated Image Contract Parity** - Align backend/frontend image payload contract so routed workflow renders real backend patterns reliably.
- [ ] **Phase 7: Verification Artifact Backfill** - Re-establish milestone verification evidence with phase verification artifacts and requirement traceability parity.
- [ ] **Phase 8: Backend Contract Suite Reconciliation** - Restore clean backend contract test guardrails so API parity confidence is evidence-backed.

## Phase Details

### Phase 1: React Pattern Generation
**Goal**: Users can fully create and choose an overlay-ready pattern inside the React generator experience.
**Depends on**: Nothing (first phase)
**Requirements**: GEN-01, GEN-02, GEN-03, GEN-04
**Success Criteria** (what must be TRUE):
  1. User can set generator inputs (aspect ratio, tile counts, tendril settings, color strategy) and immediately see validation feedback for invalid values.
  2. User can generate multiple layout variants in one action and review each variant as a selectable result.
  3. User can mark one generated variant as the active pattern that is carried forward for overlay use.
  4. User can export the selected pattern as a PNG file.
**Plans**: 3 plans

- [x] 01-react-pattern-generation-02-PLAN.md — Harden generator validation and request payload behavior
